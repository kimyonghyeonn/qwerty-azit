RDS란?
AWS 클라우드에서 관리하는 관계형 DB, 사용해보니 장점으로는 설계, 운영, 확장 및 유지보수 측면에서 가이드가 잘 되어있고 다양한 DB서비스 기능을 제공
단점은 사용해본 서비스 중 제일 비싸다는 단점이 있음

구 버전 아키텍처에서는 WEB/WAS 를 구동하는 EC2, DB를 운영하는 EC2 하나 이런 식으로 구성하여 운영하였다. 이런 구조로 운영할 경우 비용 측면에서는 저렴하지만 위와 반대로 DB 에 대한 직관적인 조작 및 DB서비스 조작이 번거롭다는 단점이 있음.

왜 RDS 로 했는가?
구 버전 아키텍처에서는 EC2 2대로 구동하여 서비스를 운영하였고, 이번에는 좀 더 대중적인 실사용 구조를 운영하기 위해 RDS 를 최저사양으로 구축해보았음

RDS 구축 가이드

1. Aurora and RDS > 데이터베이스 > 데이터베이스 생성 클릭
2. 엔진옵션 > MariaDB 선택 / 템플릿 : 개발/테스트 / 자격 증명 관리 : AWS Secrets Manager 관리
3. 인스턴스 구성 > 인스턴스 클래스 : 버스터블 클래스 (t 클래스 포함) / Instance type : db.t3.micro
4. 스토리지 : 20
5. 기본 VPC 보안 그룹 > default, 현재 WAS 가 구동중인 EC2의 보안 그룹 선택 (단, 해당 보안그룹에 RDS의 DB 포트를 허용되어 있어야함)
6. 월별 추정 요금 확인 (RDS 특성상 시간당 비용이 높게 측정되기 때문에 반드시 비용 확인 후 생성)
